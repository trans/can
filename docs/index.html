can

Server-side web components for Crystal.

can is a template language for Crystal where components are defined inside the template as a special form, not imported from a class library. Each component's <style> is scoped automatically to the elements it renders. The name is short for canned templates.

An example

This is spec/fixtures/styled_card.can verbatim:

<.def tag="card" param:title="String">
  <style>
    .card { border: 1px solid #ccc; padding: 1rem; }
    .card > h2 { color: navy; margin-top: 0; }
    .card:hover { border-color: black; }
  </style>
  <div class="card">
    <h2>{title}</h2>
    <.slot/>
  </div>
</.def>

<card title="Hello"><p>body content</p></card>

The component's .card > h2 rule only matches that card's heading — a generated data-c-card-… attribute is stamped on each element the component renders and added to every CSS selector, isolating the rule to one component without shadow DOM.
