CANNED SERVER-SIDE HTML/CSS TEMPLATES FOR CRYSTAL
GITHUB API

Can is a server-side template language for Crystal. Components are defined inside the template as a special form, their CSS is scoped automatically to their own elements, and the whole template compiles into your binary at build time — no runtime parsing, no client framework.

A template just writes HTML to an IO, so the same components render the same way whether you hand them to Kemal, Grip, plain HTTP::Server, or a build script that pipes the output to a file (like this page does).

Components, isolated

Three cards, each defined by a different component. Every one of them uses a .card class internally. They don't collide because each component's CSS is rewritten to require a generated attribute that only its own elements carry. Pop the source on any of them — same class, no juggling.

Soft

Pastel surfaces, rounded corners, a quiet voice.

One component, many variants

The same <callout> component renders three different ways, picked by a kind param. Param-driven style is just an attribute interpolation (class="callout {kind}") plus a <.if> chain inside the body.

Made of canned templates

This particular page is one static HTML file produced by crystal run docs/build.cr — every component you see was defined in a .can file and compiled at build time. The same components work unchanged inside a live HTTP handler that writes to env.response; the build script is just one way to consume the rendered output.
